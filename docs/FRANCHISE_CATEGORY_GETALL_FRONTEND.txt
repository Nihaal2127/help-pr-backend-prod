================================================================================
  Franchise category — GET list (getAll) — frontend integration
================================================================================

Base path (see server.js): /api/franchise-category

Route
-----
  GET /api/franchise-category/getAll

Authentication
--------------
  Required: Bearer JWT (same as other franchise-category routes).

  Roles allowed: Super Admin, Staff, Franchise Admin, Franchise Employee.

  Franchise Admin / Employee:
    - The server forces the list to their own franchise.
    - If they send franchise_id, it must match their user.franchise_id (403 otherwise).

  Super Admin / Staff:
    - franchise_id query parameter is required (400 if omitted).

--------------------------------------------------------------------------------
Response shape (high level)
--------------------------------------------------------------------------------

  On success (200):

  - franchise            { _id, name, admin_name, is_active }
  - categories           paginated array of global categories for that franchise
  - totalItems           total categories after filters
  - totalPages, currentPage — pagination over **categories**

  Each item in **categories** includes:

    - franchise_enabled   true if the category id is in franchise.categories[]
    - related_services    global services under that category (non-deleted)
    - global_active, franchise_active, effective_active (same semantics as services)

  No records / active_categories / inactive_categories / all_categories split.

--------------------------------------------------------------------------------
Query parameters — reference
--------------------------------------------------------------------------------

  Pagination (categories)
  -----------------------
  page, limit          defaults: page=1, limit=10

  Franchise scope
  ----------------
  franchise_id         Super Admin / Staff: required.
                       Franchise users: implied from JWT.

  Catalog filters
  ---------------
  is_active            optional true|false — filters by franchise_enabled
                       (assigned on the franchise). Omit for full catalog.

  is_request           optional true|false — when true, lists pending category
                       requests raised by users under franchise_id (same scope as
                       GET /api/category/getAll?is_request=true for that franchise).
                       When false or omitted with franchise_id, only approved global
                       catalogue rows are included (pending requests are excluded).

  Search (categories)
  -------------------
  search, q            category name first; if no name match, match related_services[].name

  Sort
  ----
  sort_by              name | id | _id  (default: name)
  sort_order, order    asc | desc (default: asc)

--------------------------------------------------------------------------------
Examples
--------------------------------------------------------------------------------

  Full catalog (franchise admin):

    GET /api/franchise-category/getAll?page=1&limit=50

  Only categories enabled on the franchise:

    GET /api/franchise-category/getAll?page=1&limit=50&is_active=true

  Super admin:

    GET /api/franchise-category/getAll?franchise_id=<FRANCHISE_OID>&page=1&limit=50

--------------------------------------------------------------------------------
getById / update
--------------------------------------------------------------------------------

  GET /get/:id and PUT /update/:id still use the legacy mapping record shape for
  toggle/reorder. Use getAll for the flat category catalog list UI.
